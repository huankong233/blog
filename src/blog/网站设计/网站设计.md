# 网站设计

## laravel

### 路由

在 `laravel11` 之后的版本需要执行命令

```bash
php artisan install:api
```

将路由在 `/routes/api.php` 中进行定义

### 控制器

```bash
php artisan make:controller
```

在类中新建函数, 函数必须使用 `public` 修饰符

### 过滤器

[关于导入](#关于导入)

```php
class ExampleController extends Controller
{
    public function register(Request $request)
    {
        $validator = Validator::make($request->post(), [
            "name" => "required|string|max:255",
            "email" => "required|email|unique:users",
            "password" => "required|string|min:6|confirmed",
            // confirmed规则默认会去验证password_confirmation字段, 如果你想要验证其他字段, 那么就需要使用same规则来验证
            // password也就是当前字段的名称, 比如密码字段叫aaa, 那么验证的字段就是 aaa_confirmation
            // "password_confirmation" => "required|string|same:password"
        ]);

        // 不过这里的一些过滤规则只能和laravel自带的blade配合使用，我们可以让他更加通用一些
        if ($validator->fails()) {
            // 判断哪个规则被触发了
            $failed = $validator->failed();

            // 判断是不是email的unique规则被触发了
            if (isset($failed["email"]["unique"])) {
                return response()->json([
                    "success" => false,
                    "message" => "邮箱已经被注册了"
                ],422);
            }

            return response()->json([
                "success" => false,
                "message" => "参数验证失败"
            ], 422);
        }

        // 如果验证通过, 那么就可以进行注册了
    }
}
```

### 模型

```bash
php artisan make:model
```

在 `model` 中有几个常用属性

```php
class User extends Model
{
    // 在插入用户的时候需要将插入的字段添加到这个属性中
    // 比如我们在调用 create 函数的时候, 传入的字段如果不在这个 $fillable 中, 那么他就不会被添加到操作数据库的sql语句中
    protected $fillable = [
        "name",
        "email",
        "password",
        "token"
    ];

    // 这里定义的属性会在序列化的时候去除
    // 比如我们通过 get 函数获取到了这个User模型, 我们直接把他放到 response->json($user) 中, 那么返回的json中就不会有token和password字段了
    protected $hidden = [
        "token",
        "password"
    ];

    // 关闭在调用 create 函数的时候添加的 created_at 和 updated_at 字段, 也就是关闭时间戳
    public $timestamps = false;
}
```

## js

### 网络请求

[Fetch MDN文档](https://developer.mozilla.org/zh-CN/docs/Web/API/Fetch_API/Using_Fetch)

使用 `fetch` 函数

在使用 `fetch` 与 `laravel` 后端进行网络请求的时候需要手动携带上 `头部信息`

```js
await fetch('/api', {
  headers: {
    'Accept': 'application/json',
    'Content-Type': 'application/json',
  },
})
```

如果 `fetch` 函数在里另一个函数里, 那么这个函数需要加上 `async` 修饰符, 并且在调用这个函数的时候需要加上 `await` 修饰符

比如:

```js
async function getData() {
  await fetch('/api', {
    headers: {
      'Accept': 'application/json',
      'Content-Type': 'application/json',
    },
  })
}

button.addEventListener('click', async () => {
  await getData()
})
```
